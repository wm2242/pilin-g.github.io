## 为什么要自建图床

市面上的免费图床要么不稳定，要么悄悄给图片加水印、压缩画质，甚至删除“违规”图片。于是我用 **Sanyue** 部署了一套完全由自己控制的图床：图片存于我的私密 Telegram 频道，通过 Cloudflare Workers 代理，再绑定个人域名。

## 测试 3：图片作为链接

图床图片不仅可以插入，还能作为可点击的链接。例如点击下方图片可以跳转到我的博客首页。

```markdown
[![点击访问博客](https://img.guoblog.dpdns.org/file/1784396388270_屏幕截图_2025-11-21_183217.png)](https://img.guoblog.dpdns.org/)
```
[![点击访问博客](https://img.guoblog.dpdns.org/file/1784396388270_屏幕截图_2025-11-21_183217.png)](https://web.guoblog.dpdns.org/)

预期：图片正常加载，点击后打开新页面。

## 测试 4：控制图片尺寸

Markdown 原生不支持设置图片尺寸，但我可以在 MDX 中混写 HTML 来实现缩放，而不会影响图床链接。

```html
<img src="https://img.guoblog.dpdns.org/file/1784396470450_a.png" width="300" alt="缩小后的图片" />
```
<img src="https://img.guoblog.dpdns.org/file/1784396470450_a.png" width="300" alt="缩小后的图片" />

如果图片以 300 像素宽度显示，说明 HTML 标签正常解析，并且图床支持任意尺寸的调用。

## 测试 5：多图并排展示

有时候需要在博客中展示多张截图对比，我使用 HTML 表格来实现并排图片。

```html
<table>
  <tr>
    <td><img src="https://img.guoblog.dpdns.org/file/1784396108034_am1.png" width="200" /></td>
    <td><img src="https://img.guoblog.dpdns.org/file/1784396388270_屏幕截图_2025-11-21_183217.png" width="200" /></td>
  </tr>
</table>
```

## 测试 7：管理能力——删除与失效

这张图片是我临时上传的，随后我会在 Telegram 频道中**删除这条消息**。

![即将被删除的图片](https://img.guoblog.dpdns.org/file/1784396470450_a.png)

删除后再刷新博客，上方应该出现一张裂图（或者直接不显示）。这证明我对图床拥有完全控制权，能够随时移除不当内容。
